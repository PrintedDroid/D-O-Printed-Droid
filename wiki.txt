<h1>D-O Droid Control System v2.1 – Setup and Configuration Guide</h1>

The D-O Droid control system version 2.1 is a complete all-in-one solution for Arduino Mega that handles self-balancing, sound control, servo control, and all RC functions.

<strong>Available Versions:</strong>
<ul>
<li><strong>v1.1:</strong> Basic iBus control (simple, fewer features)</li>
<li><strong>v2.1:</strong> Full featured with CLI configuration (RECOMMENDED)</li>
<li><strong>v3.x:</strong> Universal controller with PWM/iBus modes and watchdog</li>
<li><strong>ESP32-S3:</strong> Advanced version with TFT display and onboard IMU</li>
</ul>



<h2 id="why-ibus">Why Choose iBus v2.1?</h2>

The iBus v2.1 is a complete system that combines ALL functions into one Arduino Mega:

<ul>
<li>MPU6050/6500/9250/6886 IMU-based self-balancing with PID control</li>
<li>10-channel iBus RC receiver support</li>
<li>DFPlayer Mini sound system with personality features</li>
<li>Multi-servo control for head and mainbar movement</li>
<li>Battery monitoring with low voltage protection</li>
<li>Runtime configuration menu via Serial</li>
<li>Idle animations and state reactions</li>
<li>RC Mixing Mode selection (Arcade/Tank)</li>
</ul>

<strong>One Mega does everything - no additional controllers needed!</strong>



<h2 id="features">Major Features</h2>

<h3>1. Runtime Configuration Menu (CLI)</h3>

No more editing code and re-uploading! All settings can be changed via Serial Monitor:

<pre><code>=== CONFIGURATION MENU ===
1. PID Configuration
2. Feature Toggles
3. Battery Settings
4. Sound Settings
5. Driving Dynamics
6. Adaptive PID
7. Save &amp; Exit
8. Factory Reset
9. Show Current Values</code></pre>

<strong>Access:</strong> Send 'm' via Serial Monitor at any time

<h3>2. RC Mixing Mode Selection (NEW in v2.1.2)</h3>

Choose between two control styles:

<strong>ARCADE MODE (Default - Recommended):</strong>
<ul>
<li>CH1 (Right Stick Left/Right) = Steering</li>
<li>CH2 (Right Stick Up/Down) = Throttle (Forward/Backward)</li>
<li>One stick controls everything - intuitive like a video game</li>
<li>Best for standard FlySky transmitter configuration</li>
</ul>

<strong>TANK MODE (Original):</strong>
<ul>
<li>CH1 = Left Motor speed</li>
<li>CH2 = Right Motor speed</li>
<li>Each stick controls one motor directly</li>
<li>Requires mixing on transmitter or custom channel setup</li>
</ul>

<strong>Configure via:</strong> CLI Menu → 5. Driving Dynamics → RC Mixing Mode

<h3>3. Multiple Setup Modes</h3>

<pre><code>// Mode 0: PWM Only (balance only, external sound controller)
// Mode 1: Hybrid (iBus drive + PWM sound)
// Mode 2: Pure iBus (RECOMMENDED - all in one)</code></pre>

<h3>4. Battery Monitoring (Optional)</h3>

Monitors 2S+2S LiPo configuration (6.0V - 8.4V range)

<div style="background-color: #fff3cd; padding: 15px; border-left: 4px solid #ffc107; margin: 15px 0;">
<strong>IMPORTANT:</strong> Battery monitoring is OFF by default!
Only enable if you have the voltage divider circuit connected.

<ul>
<li>Without voltage divider: Pin A15 floats and shows random values</li>
<li>With voltage divider: Accurate battery percentage display</li>
</ul>

<strong>Enable via:</strong> CLI Menu → 2. Feature Toggles → Battery Monitor
</div>

<h3>5. Adaptive PID Control</h3>

Automatically adjusts PID parameters based on driving speed:

<ul>
<li><strong>Slow</strong> (standing/creeping): Higher Kp for stability</li>
<li><strong>Medium</strong> (normal driving): Balanced settings</li>
<li><strong>Fast</strong> (quick maneuvers): Lower Kp, responsive handling</li>
</ul>

<h3>6. Advanced Driving Dynamics</h3>

<ul>
<li>Motor ramping for smooth acceleration</li>
<li>Exponential RC curves for precise control</li>
<li>Dynamic target angle (leans into movement)</li>
<li>Turn compensation</li>
<li>Acceleration limiting</li>
<li>Configurable deadband</li>
</ul>

<h3>7. Personality Features</h3>

D-O comes alive with:
<ul>
<li>Random idle sounds and head movements after 3 seconds</li>
<li>State reactions (tilt warnings, recovery sounds)</li>
<li>Mood-based sound selection via RC channel</li>
</ul>

<h2 id="setup">Initial Setup Process</h2>

<h3>Step 1: First Upload</h3>

<ol>
<li>Open <code>D_O_printed_droid_rc_ibus_v2.1.ino</code> in Arduino IDE</li>
<li>Select Board: Arduino Mega 2560</li>
<li>Select correct COM port</li>
<li>Upload sketch</li>
<li>Open Serial Monitor (9600 baud)</li>
<li>You'll see: <code>"Send 'c' for IMU calibration or 'm' for menu within 3 seconds..."</code></li>
</ol>

<h3>Step 2: IMU Calibration (CRITICAL!)</h3>

<ol>
<li>Place D-O in balanced upright position on flat surface</li>
<li>Send <code>'c'</code> within 3 seconds of boot</li>
<li>Keep D-O <strong>PERFECTLY STILL</strong> during calibration (5 seconds)</li>
<li>"Calibration saved" confirms success</li>
<li>Settings auto-save to EEPROM</li>
</ol>

<em>Note: Re-calibrate if balance seems off or after moving the IMU</em>

<h3>Step 3: Configuration Menu</h3>

Send <code>'m'</code> via Serial Monitor at any time to access the menu:

<pre><code>=== D-O Configuration Menu ===
1. PID Configuration
2. Feature Toggles
3. Battery Settings
4. Sound Settings
5. Driving Dynamics
6. Adaptive PID
7. Save &amp; Exit
8. Factory Reset
9. Show Current Values</code></pre>

<strong>Example - Setting up RC Mixing Mode:</strong>
<ol>
<li>Send <code>'m'</code> to enter menu</li>
<li>Send <code>'5'</code> for Driving Dynamics</li>
<li>Follow prompts to configure mixing mode</li>
<li>Send <code>'7'</code> to Save &amp; Exit</li>
</ol>

<h3>Step 4: Selecting Setup Mode</h3>

For new builds, select Pure iBus mode (Mode 2):

<pre><code>--- Setup Type Configuration ---
0 = PWM Only (Balance only, external controller for sound)
1 = Hybrid (iBus drive + PWM sound on separate pins)
2 = Pure iBus (RECOMMENDED - all features in one)</code></pre>

<h3>RC Mixing Modes Explained</h3>

<strong>ARCADE MODE (mixing_mode = 1, DEFAULT):</strong>
<ul>
<li>CH1 = Steering (left/right stick movement)</li>
<li>CH2 = Throttle (forward/backward stick movement)</li>
</ul>

<pre><code>Motor calculation:
  Left Motor  = Throttle + Steering
  Right Motor = Throttle - Steering</code></pre>

<em>Best for: Standard FlySky setup, intuitive control</em>

<strong>TANK MODE (mixing_mode = 0):</strong>
<ul>
<li>CH1 = Left Motor speed directly</li>
<li>CH2 = Right Motor speed directly</li>
</ul>

<em>Best for: Custom transmitter mixing, advanced users</em>

<h2 id="troubleshooting">Troubleshooting</h2>

<h3>"No DFPlayer found" / No Sound</h3>
<ul>
<li>Check wiring, especially the 1k resistor on TX line</li>
<li>Verify SD card is FAT16/FAT32 formatted</li>
<li>Ensure files are named correctly (0001.mp3, not 1.mp3)</li>
<li>Check speaker connection (SPK1 and SPK2)</li>
<li>Try different SD card</li>
</ul>

<h3>Poor Balance / Tips Over Immediately</h3>
<ol>
<li>Run IMU calibration (<code>'c'</code> command at boot)</li>
<li>Check that D-O is placed upright during calibration</li>
<li>Adjust PID values via menu:
<ul>
<li>Oscillating rapidly: Decrease KP, increase KD</li>
<li>Too wobbly/slow response: Increase KP</li>
<li>Drifting slowly: Small increase to KI</li>
</ul></li>
<li>Default values: KP=25, KI=0, KD=0.8</li>
<li>Check motor wiring (both should spin same direction for forward)</li>
</ol>

<h3>No RC Response</h3>
<ul>
<li>Verify iBus wire connected to Pin 19 (RX1)</li>
<li>Check transmitter/receiver binding (solid LED on receiver)</li>
<li>Ensure receiver is powered (5V from Mega or BEC)</li>
<li>Verify iBus output is enabled on receiver</li>
<li>Check if Setup Mode is correct (Mode 2 for Pure iBus)</li>
</ul>

<h3>Steering Too Fast / Driving Too Slow</h3>
<p>This is likely a Mixing Mode issue:</p>
<ol>
<li>Enter CLI menu (<code>'m'</code>)</li>
<li>Go to Driving Dynamics (option 5)</li>
<li>Check current Mixing Mode</li>
<li>ARCADE mode (1): CH2=throttle, CH1=steering - RECOMMENDED</li>
<li>TANK mode (0): CH1=left motor, CH2=right motor</li>
</ol>

<h3>Battery Shows Wrong Voltage (1.8V, 2.5V, random values)</h3>
<ul>
<li>Battery monitoring reads A15 pin</li>
<li>Without voltage divider connected, pin floats = random values</li>
<li><strong>Solution 1:</strong> Disable battery monitoring (Menu → Feature Toggles)</li>
<li><strong>Solution 2:</strong> Connect proper voltage divider circuit</li>
</ul>

<h3>Servos Jittering or Not Moving</h3>
<ul>
<li>Servos need separate power supply (5-6V BEC)</li>
<li>DO NOT power servos from Mega 5V pin (not enough current)</li>
<li>Only signal wire goes to Mega GPIO</li>
<li>Check servo pin assignments match your wiring</li>
</ul>

<h3>"EMERGENCY STOP" Triggers Immediately</h3>
<ul>
<li>D-O tilted more than 45 degrees at startup</li>
<li>IMU not calibrated or miscalibrated</li>
<li>IMU mounted incorrectly (check orientation)</li>
<li>Run calibration with D-O perfectly upright</li>
</ul>

<h3>Configuration Menu Options</h3>

<strong>1. PID Configuration</strong>
<ul>
<li>KP (Proportional): 0-100, default 25</li>
<li>KI (Integral): 0-10, default 0</li>
<li>KD (Derivative): 0-10, default 0.8</li>
<li>Target Angle: -10 to +10, default varies</li>
</ul>

<strong>2. Feature Toggles</strong>
<ul>
<li>Motor Ramping: Smooth acceleration (ON/OFF)</li>
<li>Adaptive PID: Speed-based PID adjustment (ON/OFF)</li>
<li>Dynamic Angle: Lean into movement (ON/OFF)</li>
<li>Idle Actions: Random sounds/movements (ON/OFF)</li>
<li>State Reactions: Tilt warnings etc. (ON/OFF)</li>
<li>Battery Monitor: Voltage monitoring (ON/OFF) - <strong>OFF by default!</strong></li>
</ul>

<strong>3. Battery Settings</strong>
<ul>
<li>Warning Voltage: Default 6.8V</li>
<li>Critical Voltage: Default 6.4V</li>
<li>Voltage Divider Factor: Calibration value</li>
</ul>

<strong>4. Sound Settings</strong>
<ul>
<li>Volume: 0-30, default 25</li>
<li>Min Sound Interval: Prevents overlap</li>
<li>Idle Interval Min/Max: Random idle timing</li>
</ul>

<strong>5. Driving Dynamics</strong>
<ul>
<li>Ramp Rate: 0.01-1.0, acceleration smoothing</li>
<li>Max Acceleration: Speed change limit</li>
<li>Max Lean Angle: Forward lean limit</li>
<li>RC Deadband: Center dead zone</li>
<li>Expo Factor: Stick response curve</li>
<li>Mixing Mode: 0=Tank, 1=Arcade (default)</li>
</ul>

<strong>6. Adaptive PID</strong>
<ul>
<li>Slow/Medium/Fast KP values</li>
<li>Slow/Medium/Fast KD values</li>
</ul>



<h2 id="upgrade">Upgrading from Older Versions</h2>

<h3>From v1.1 to v2.1</h3>
<ol>
<li>Upload v2.1 sketch</li>
<li>Run IMU calibration (<code>'c'</code>)</li>
<li>Configure via menu (<code>'m'</code>)</li>
<li>All settings saved to EEPROM</li>
</ol>

<h3>From v2.0 to v2.1</h3>
<ol>
<li>Upload v2.1 sketch</li>
<li>Previous EEPROM settings should be compatible</li>
<li>New features (Mixing Mode) will use defaults</li>
<li>Verify settings via menu</li>
</ol>



<h2 id="best-practices">Best Practices</h2>

<ol>
<li>Always calibrate IMU on perfectly flat, stable surface</li>
<li>Start with default PID values before tuning</li>
<li>Use Arcade mixing mode for easiest control</li>
<li>Keep battery monitoring OFF unless voltage divider is connected</li>
<li>Test balance without head first, then add head</li>
<li>Use Pure iBus mode (2) for all features in one controller</li>
<li>Save configuration after changes (Menu option 7)</li>
</ol>
